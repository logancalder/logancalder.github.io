Tuve el placer de competir en el Hack 4 Humanity 2025 de SCU.

El proyecto de nuestro equipo para este hackathon fue lmk, un servicio web basado en IA y Next.js que podía analizar eficazmente situaciones peligrosas y alertar a los usuarios respectivos en las proximidades.

Nuestro proyecto fue inspirado por preocupaciones éticas como el efecto espectador y el sesgo de reporte. La gente tiende a dudar en reportar incidentes, asumiendo que alguien más lo hará, lo que a menudo lleva a problemas sin resolver. En algunas comunidades, los incidentes graves pueden ser minimizados o ignorados debido a presiones sociales. Nos motivamos aún más a desarrollar una solución debido a los recientes (al momento de escribir esto) incendios en Pacific Palisades y la región general de Los Ángeles.

lmk es una plataforma diseñada para reducir la barrera para reportar peligros, proporcionando visibilidad a incidentes locales. Trabajando para mantener seguras las comunidades, este proyecto está diseñado para ser completamente colaborativo, permitiendo que el público mantenga completamente los datos del servicio.

El flujo de trabajo de nuestro proyecto involucró un frontend increíblemente bien pulido vinculado con un potente backend y base de datos. Utilizamos Next.js, TailwindCSS, ShadCN, y Mapbox para desarrollar una interfaz de aspecto profesional, completa con animaciones, un mapa interactivo y formularios para enviar. El backend respaldó esta interfaz de usuario con Next.js, API de Mapbox, API de OpenAI, y Supabase (una base de datos PostgreSQL).

Mientras que algunas tareas se pudieron completar con simple recuperación de JSON, utilizamos GPT 4 y 4-Turbo de OpenAI para desarrollar una colección intelectualmente fuerte de agentes de IA, trabajando juntos para poblar plantillas JSON prediseñadas. Estos datos luego se enviaban a Supabase, de donde podíamos obtener los datos para mostrarlos en el mapa o en informes.

También diseñamos este proyecto pensando en que fuera una startup, por lo que la autenticación de usuarios a través de cuentas de Google y Supabase proporcionó la muy necesaria autorización y seguridad para nuestra aplicación. Dado que esto está implementado, no sería difícil implementar funciones de pago, permitiendo que este proyecto se convierta en un negocio.

Los planes futuros para este proyecto también incluyen la integración con redes sociales, a través de la cual escanearíamos publicaciones para detectar áreas dañinas o peligrosas y advertir apropiadamente a los usuarios. Combinado con esto, podríamos enviar mensajes SMS a los usuarios si habilitan compartir ubicación con nosotros, permitiéndonos advertirles sin necesidad de abrir la aplicación. Como optimización, también podríamos revisar nuestros agentes de IA para trabajar en paralelo en lugar de concurrentemente, para mejorar significativamente nuestro tiempo de envío (~ 50% de reducción en tiempo de ejecución).
